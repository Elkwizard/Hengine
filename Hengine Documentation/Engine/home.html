Engine

The Engine class is used to create the Engine Instance used in the project. The engine is the 'g' or 'Game' property of the Project. The engine manages timing and management of update loops and Scenes.


Properties:

  • fps

    A float representing the current frames per second that the game is running at.

  • frameCount

    An integer showing how many frames have passed since the page loaded.

  • console

    A Console object. This property will only be available if the 'Console.js' script is enabled.

  • paused

    A boolean indicating whether or not the engine is fully paused.

  • renderer

    A reference to the Artist instance used by the Project. This acts just the normal "c" or "Renderer" object.
    Example:g.renderer.draw('black').rect(50, 50, 100, 50); // black rectangle is drawn

  • scene

    A Scene object representing the Scene instance used by the Project. This can be used to run standard management methods.
    Example:let player = g.scene.addRectElement("player", 50, 50, 50, 50); // 50 by 50 physics element is created

  • beforeScript

    A Script object. Script.run will be called on the script every frame before the Scene is updated.

  • updateScript

    A Script object. Script.run will be called on the script every frame just before the frame is drawn.

  • afterScript

    A Script object. Script.run will be called on the script every frame after the Scene has completed updating.

  • f

    This property will only exist if the 'Helper Functions.js' script is enabled. Engine.f is a FunctionLibrary object containing helpful functions.


Methods:

  • constructor(String wrapperID, int width, int height, float airResistance, float gravity, String canvasID)

    Creates an Artist object with a width of width, a height of height, both measured in CSS 'px'. The Artist object's canvas is either the element associated with canvasID, if provided, and otherwise, a HTMLCanvasElement is created to be used. If wrapperID is provided, then the instantiated canvas will be appended to the inside of the element associated with wrapperID, otherwise, the canvas will be appended to the document.body. The Artist will then be stored in Engine.renderer. A Scene object is then instantiated with parameters ('Engine Scene', Engine.renderer, gravity, airResistance, Engine). The Scene object is stored in Engine.scene.

  • beforeUpdate()[1]

    This method is called before the Engine has started updating.

  • update()[1]

    This method is called before the Scene instance is updated.
    Example:g.update = () => {alert(g.frameCount)}; //alerts the number of frames that have passed once per frame

  • afterUpdate()[1]

    This method is called after the entire Engine has finished updating after a frame.

  • fixedUpdate()[1]

    This method is called every time physics calculations take place, This is usually about every 16ms.

  • clear()

    This method is called every frame to clear the display before the frame is drawn.
    Example:g.afterUpdate = g.clear; //nothing will ever appear onscreen

  • animate()

    This method is called every frame and calls Engine.beforeUpdate, Engine.update, and Engine.afterUpdate. It also handles rendering.

  • engineUpdate()

    This method is called in a loop every 16ms. It manages physics and calls Engine.fixedUpdate.

  • pause()

    Prevents the Engine.animate and Engine.engineUpdate being called.

  • play()

    Undoes Engine.pause.

  • setTitle(String title)

    Sets the name of the tab to title.

  • getDistance(float x1, float y1, float x2, float y2)

    returns the distance in 2D space between the points (x1, y1) and (x2, y2).

  • getDistanceWithPoints(Vertex p1, Vertex p2)

    returns the distance in 2D space between the points p1 and p2.

  • extend(float a, float b)

    returns the a extended away from zero by b.
    Example:g.console.log(g.extend(-5, 6)); // -11

  • contract(float a, float b)

    returns the a moved towards zero by b.
    Example:g.console.log(g.contract(-5, 6)); // 1

  • [1]: this method starts empty. its purpose is to be set by the user.
