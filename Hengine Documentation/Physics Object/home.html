PhysicsObject & CirclePhysicsObject

PhysicsObject inherits from SceneObject. PhysicsObjects are used in the Scene to represent colliding gameplay elements. PhysicsObjects would be things like players or the ground. The circle varient of this class (CirclePhysicsObject) is the same except x and y represent the center coordinates and there is a radius property that stores the radius of the CirclePhysicsObject.

Properties:

  • cullGraphics

    A boolean indicating whether or not the PhysicsObject should be graphics culled if Scene.cullGraphics is true.

  • applyGravity

    A boolean indicating whether or not the PhysicsObject can fall.

  • slows

    A boolean indicating whether or not the PhysicsObject slows down over time.

  • canCollide

    A boolean indicating whether or not the PhysicsObject can collide.

  • shown

    A boolean indicating whether or not the PhysicsObject is in the part of the Scene that is onscreen.

  • maxSpeedX

    The maximum speed at which the PhysicsObject can travel horizontally.

  • maxSpeedY

    The maximum speed at which the PhysicsObject can travel vertically.

  • response.collide

    An Object containing 'top', 'bottom', 'left', 'right', and 'general' properties containing functions which will be called when the PhysicsObject collides in the corresponding directions.

  • colliding

    An Object containing 'top', 'bottom', 'left', 'right', and 'general' properties with values that are either null/false or an array of colliding objects, indicating whether or not the PhysicsObject is currently colliding with something on that side.

  • allCollidingWith

    An Object with two methods:

    • includes(String name)

      Returns false if the PhysicsObject is not colliding with the element with a name of name. Otherwise, returns the element.

    • includesTag(String tag)

      Returns false if the PhysicsObject is not colliding with the element with a tag of tag. Otherwise, returns the element.

  • speed

    A Vector2 representing the speed of the PhysicsObject.

  • accel

    A Vector2 representing the acceleration of the PhysicsObject.


Methods:

  • constructor(String name, float x, float y, float width, float height, boolean applyGravity, Controls controls, String tag, Scene home)

    Calls SceneObject.prototype.constructor with parameters (name, x, y, width, height, applyGravity, controls, tag, home). Whether or not the PhysicsObject has gravity applied is determined by applyGravity.

  • collideBasedOnRule(PhysicsObject el)

    Is called to determine whether the PhysicsObject and el should collide. This method is meant to be set by the user. It is only called after it is determined that the two PhysicsObjects are overlapping.

  • optimize(PhysicsObject el)

    This method is called as a preliminary check before collisions detection where el is the PhysicsObject being checked. This method is meant to be set by the user but it starts as a distance check. PhysicsObject.optimize should return a boolean.

  • stop()

    causes the PhysicsObject to stop moving entirely.

  • moveTowards(Vertex target, float ferocity)

    causes the PhysicsObject move towards target at a speed of ferocity.

  • moveAwayFrome(Vertex target, float ferocity)

    causes the PhysicsObject away from target at a speed of ferocity.

  • removeCollisions()

    Prevents the PhysicsObject from colliding.

  • allowCollisions()

    Allows the PhysicsObject to collide with other PhysicsObjects.

  • removeGravity()

    Prevents the PhysicsObject from falling.

  • allowGravity()

    Allows the PhysicsObject to fall.

  • applyForce(String dir, float amount)

    Adds amount force in the dir direction.

  • setForce(String dir, float amount)

    Sets the speed in direction dir to amount.

  • collide(PhysicsObject el)

    returns true if el and the PhysicsObject are overlapping, false, otherwise.
