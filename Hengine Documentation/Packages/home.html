Packages

Purpose

When using the Hengine, different projects are separated into folder structures called Packages. Packages contain the Environment required to properly use the Hengine. The Package is where your sprites, animations, sounds, and source code are stored. The Hengine itself is not stored inside the Package.


Structure

Packages only work if they have the correct structure. For Project.loadImage and Project.loadSound to work, the Package's files have to be the correct locations.
Anatomy of a Hengine Package: 

				packageName*─┬──Source─┬──Source.js
				             │         ├──Set List.js
				             │         ├──Scripts.js
				             │         ├──Project.html
				             │         └──Offline Project.html
				             │
				             ├──Sound─┬──SoundFile.mp3**
				             │        └──SoundFile2.mp3**
				             │
				             ├──Art─┬──Sprites─┬──ImageFile.png***
				             │      │          └──ImageFile2.png***
				             │      │
				             │      └──Animations──animation1****─┬──1.png*****
				             │                                    └──2.png*****
				             │
				             ├──Launch.html
				             │
				             ├──Offline Launch.html
				             │
				             └──favicon.ico

* This can be whatever you like. If you never use Engine.setTitle, then this will be the title of the page.
** These can be any sound files with any names and extensions.
*** These can be any image files with any names and extensions.
**** This folder represents your animations. You can have any number of these and they can have any names.
***** These are the frames in your animation. They have to be named according to their frame number and the must be .pngs.

The Default Package starts with 'Chicken.png' in the Sprites folder and a 'Chicken Flap' animation in the Animations folder. All of your Hengine Packages should be stored inside of a 'Packages' Folder.


Project.html & OfflineProject.html

Project.html and OfflineProject.html are the HTML files can be opened to actually run your project. These files can be found in the Source folder of your Package. Project.html needs an internet connection to run, because it needs to access the Engine. OfflineProject.html does not need an internet connection but it will only run with the version of Hengine available when you first downloaded the package. Now, you can use OfflineProject.html to run your game on your computer.
Project.html and OfflineProject.html have a similar structure:

  • Rendering

    This section contains the two scripts used for rendering, 'Canvas Complete.js' and 'Color.js'

  • Optional

    This contains the optional scripts:

    • Helper Functions.js

      Enables the FunctionLibrary class.

  • Required

    The rest of the required pieces of the Engine.

You can comment out the scripts in the optional section to make load times faster, and you can enable them again to get the additional classes. For offline and online, the Launch.html files are mostly blank html files that have the game embedded in them. These can be used for making webpages with the Game inside.
