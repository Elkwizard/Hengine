class PARTICLE_SPAWNER extends ElementScript
Adds particle emitting functionality to a SceneObject. All properties of this class are read-only. const particles = scene.main.addElement("particles", width / 2, height / 2); particles.scripts.add(PARTICLE_SPAWNER, { delay: 1, lifeSpan: 100, init(particle) { particle.velocity = Vector2.fromAngle(Random.angle()).times(Random.range(2, 5)); particle.data.size = Random.range(2, 5); }, draw(renderer, particle) { renderer.draw(new Color("black")).circle(particle.position, particle.data.size); } });
Properties
particleInitParticle => void
The function that is called to initialize particles. This will be passed the particle object for each particle created. This uses the alternate key properties.init when provided in a parameter. Default is a no-op
particleUpdateParticle => void
The function that is called to update particles each frame. This will be passed each particle object each frame. Since this function is not culled, all non-rendering logic should be here. This uses the alternate key properties.update when provided in a parameter. Default is a no-op
particleDraw(Artist, Particle) => void
The function that is called to render particles each frame. This will be passed an Artist and a particle object for each particle object on-screen each frame. This uses the alternate key properties.draw when provided in a parameter. Default is a no-op
slowsBoolean
Whether or not particles will have air resistance applied. Default is false
fallsBoolean
Whether or not particles will have gravity applied. Default is false
activeBoolean
Whether or not particles will be spawned passively over time. Default is true
delayNumber
The delay (in frames) between particle spawns. This can be less than 1. Default is 1
lifeSpanNumber
The duration (in frames) of each particle's lifetime. Default is 100
radiusNumber
The effective radius of each particle used to compute culling. This does not affect the appearance of the particles. Default is 10
imageTypeClass
This specifies how particles should be rendered. If this is FastFrame, they will be rendered on a separate surface and then be copied over. If this is CanvasImage, they will be rendered directly to the screen. This is not an actual property and can only be specified as a property of a properties argument. Default is FastFrame
