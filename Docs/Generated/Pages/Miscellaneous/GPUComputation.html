class GPUComputation
Represents an operation that can be executed in parallel with itself on the GPU. The main entry point is: int[OUTPUT_BYTES] compute(int[INPUT_BYTES] inputs) { // your main code here } Where INPUT_BYTES and OUTPUT_BYTES are the sizes of the inputs and outputs of the operation. There are some helper variables and functions provided, specifically: #define PROBLEM_INDEX // the index of the operation in the batch #define PROBLEMS // the size of the batch // returns the input data for a specific operation in the batch int[INPUT_BYTES] getProblem(int index);

compute(input, output?)ByteBuffer
Evaluates the operation on a batch of inputs simultaneously. Returns the outputs in a contiguous buffer in the same order as the inputs.
Parameters
input ByteBuffer
The buffer containing the inputs of all the operations. These should be directly contiguous
output ByteBuffer
The buffer to write the output of the operations to. It will be packed contiguously. If not specified, this will be a newly created buffer

GPUComputation.destructure(glsl)String
Converts GLSL which uses structs for the parameter and return value of the compute() function to GLSL that can be used in the GPUComputation constructor. The order of the struct data in the input and output buffers is the same as the declaration order of their fields. When using this, getProblem() will return the correct struct type.
Parameters
glsl String
The GLSL source code using structs for the input and output of the compute() function
