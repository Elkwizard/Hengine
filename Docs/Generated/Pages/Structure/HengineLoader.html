class HengineLoader
Allows for the loading of both the rest of the Hengine and additional external files. The file containing this class HengineLoader.js is the only file that needs to be loaded directly to use the Hengine. Other files can be loaded via the HengineLoader's API. The web URL for this file is: https://elkwizard.github.io/Hengine/Package/Engine/Manage/HengineLoader.js The engine will be loaded in 3D Mode if the URL query parameter ?3d is appended to the end of the HengineLoader.js path. This class is a singleton, and the single instance can be accessed via a static property. async function load() { // choose which file to load based on an external file await HengineLoader.load([ new HengineTextResource("whichToLoad.txt") ], false); // load the selected script const fileName = loadResource("whichToLoad.txt"); await HengineLoader.load([ new HengineScriptResource(`${fileName}.js`) ]); }

Methods
loadResource(src)Any/null
Retrieves a specific resource. If the resource failed to load, this returns null. This method is also available on the global object. If the resource implements Copyable, a copy of the resource will be returned.
Parameters
src String
An arbitrarily-lengthed tail end of the source of the resource. This can be as few characters as are needed to be unambiguous, or may be the entire path
loadTexture(src)Sampler/null
Retrieves a specific image resource as a Sampler for use in 3D rendering. If the resource failed to load, this returns null. This method is also available on the global object.
Parameters
src String
An identifying portion of the image's source. This behaves like the src parameter in .loadResource()
HengineLoader.load(userResources, done?)Promise
Loads a series of resources, and optionally starts the update loop after the loading completes. If the rest of the Hengine has yet to be loaded, it will be loaded before any of the resources passed to this function. Returns a promise that resolves to the HengineLoader instance when all the resources are loaded.
Parameters
userResources HengineResource[]
The resources to load
done Boolean
Whether or not the update loop should start after the resources are loaded. Default is true
