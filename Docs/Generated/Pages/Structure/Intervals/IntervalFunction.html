class IntervalFunction
A wrapper for operations that happen over time or after a time. These can generally be created by methods of IntervalManager. const transitionDone = intervals.transition(t => { console.log("Progress: " + t); }, 5); transitionDone.then(() => { console.log("The transition has completed"); }); // Progress: 0 // Progress: 0.2 // Progress: 0.4 // Progress: 0.6 // Progress: 0.8 // Progress: 1 // The transition has completed
Subclasses
DelayedFunction, TransitionFunction, ContinuousFunction, WaitUntilFunction
Properties
fn Function
The function to call during the operation
type Symbol
When during the update cycle the function updates
promise Promise
A promise which resolves when the operation completes
done Boolean
Indicates whether the operation has completed
timer Number
The amount of frames the IntervalFunction has existed for
interval Number
The total duration of the operation. The operation will complete after the timer exceeds this value
IntervalFunction.BEFORE_UPDATE Symbol
This symbol indicates that the operation should take place before the screen is cleared
immutable
IntervalFunction.UPDATE Symbol
This symbol indicates that the operation should take place immediately before the main engine update
immutable
IntervalFunction.AFTER_UPDATE Symbol
This symbol indicates that the operation should take place immediately after the main engine update
immutable
class DelayedFunction extends IntervalFunction
This IntervalFunction executes an operation once after a specified delay.
class TransitionFunction extends IntervalFunction
This IntervalFunction executes an operation every frame over a duration and is passed a completion percentage.
class ContinuousFunction extends IntervalFunction
This IntervalFunction executes an operation once every frame forever.
class WaitUntilFunction extends IntervalFunction
This IntervalFunction executes an operation once after a provided condition is met. After the condition is met, the WaitUntilFunction finishes.
