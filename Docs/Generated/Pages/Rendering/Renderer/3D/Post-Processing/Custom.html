class Artist3D.CustomEffects extends Artist3D.PostProcess
Represents a collection of user-defined post-processing effects. Effects can be defined using a full-screen GLSL program, which will be provided with information about the current state of the rendering. This class should not be constructed, and should instead be accessed via the .before and .after properties of an Artist3D.PostProcessEffects. // chromatic abberation effect renderer.postProcess.after.addEffect("abberation", ` vec4 shader() { vec2 resolution = vec2(textureSize(colorTexture, 0)); vec2 position = resolution * uv; return vec4( texture(colorTexture, (position + vec2(-3, 0)) / resolution).r, texture(colorTexture, (position + vec2(0, 3)) / resolution).g, texture(colorTexture, (position + vec2(3, 0)) / resolution).b, 1 ); } `);
Methods
reorder(order)void
Re-orders the way in which the effects are applied.
Parameters
order String[]
The names of the effects to be applied, in the order to apply them. Any names omitted will be removed from the list of effects
addEffect(name, glsl, uniforms?)void
Adds a new full-screen post-processing effect after the existing effects. The shader program will be provided with several inputs, specifically the following:
InputDescription
vec2 uvThe UV coordinates of the current pixel in screen space
sampler2D colorTextureThe current color content of the screen
sampler2D depthTextureThe current depth content of the screen
Lens lensThe lens being used for rendering, with the properties of a JS Lens object, including .camera
Parameters
name String
The name of the new effect
glsl String
The source code for the full-screen fragment shader effect. This must define a vec4 shader() entry-point function
uniforms Object
An object containing uniform values to pass to this effect. Changes to the object will be reflected in the uniform values. Default is an empty object
removeEffect(name)void
Removes the effect with a given name.
Parameters
name String
The name of the effect to remove
