class Texture extends ImageType implements Copyable, Serializable
Represents a 2D grid of pixels which can be directly accessed and modified. const texture = new Texture(300, 300); // create a voronoi texture texture.shader((x, y, dest) => { const intensity = Random.voronoi2D(x, y, 0.1); dest.set(Color.grayScale(intensity)); });
Properties
dataByteBuffer
The pixel data of the texture. Modifying this buffer will modify the texture
loopsBoolean
Whether or not pixel coordinate parameters to methods will be wrapped around the edges of the texture space
Methods
get brightnessNumber[][]
Returns a 2D array of the brightness values for all pixels in the texture. The first index is the x coordinate, the second the y.
get redNumber[][]
Returns a 2D array of the red channel values for all pixels in the texture. The first index is the x coordinate, the second the y.
get greenNumber[][]
Returns a 2D array of the green channel values for all pixels in the texture. The first index is the x coordinate, the second the y.
get blueNumber[][]
Returns a 2D array of the blue channel values for all pixels in the texture. The first index is the x coordinate, the second the y.
get alphaNumber[][]
Returns a 2D array of the alpha channel values for all pixels in the texture. The first index is the x coordinate, the second the y.
clear()void
Clears the texture to contain only transparent black pixels.
validPixel(x, y)Boolean
Checks whether the given coordinates are valid pixel coordinates. This method ignores potential coordinate wrapping.
Parameters
x Number
The x coordinate to check
y Number
The y coordinate to check
getPixel(x, y, valid?)Color
Returns a reference to the color of the pixel at a specific location. This color object is managed by the texture, and will only be valid until the next call to this method. If the color data is needed more permanently, create a copy of the return value.
Parameters
x Number
The x coordinate of the pixel
y Number
The y coordinate of the pixel
valid Boolean
Whether or not the pixel coordinates are known to be valid. Default is false
shader(mapping)Texture
Applies an in-place mapping to every pixel in the texture. Returns the caller.
Parameters
mapping (Number, Number, Color) => void
A shader function called for every pixel. The return value of this function, but it takes in three arguments: the x and y coordinates of the pixel, and the pixel color. Modifying the state of the pixel color argument will change the pixel color in the texture
setPixel(x, y, color, valid?)void
Changes the color of a specified pixel in the texture.
Parameters
x Number
The x coordinate of the pixel
y Number
The y coordinate of the pixel
color Color
The new color of the pixel
valid Boolean
Whether or not the pixel coordinates are known to be valid. Default is false
blur(radius)Texture
Applies an in-place box blur with a specified radius to the texture. Returns the caller.
Parameters
radius Number
The radius of the box blur

Texture.fromImageType(image | image, region | image, x, y, width, height)Frame
Returns a texture containing the (optionally clipped) contents of an image. If no clipping parameters are provided, the whole image will be copied. The copy will have the same dimensions as the original, so if the original has a pixel ratio greater than 1, this operation will result in a loss of detail.

The image to copy data from

width Number
The width of the region
height Number
The height of the region

Creates a new grayscale texture based on a 2D grid of brightness values.
